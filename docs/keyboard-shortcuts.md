# The Write — キーボードショートカット

OS を自動判定し、Mac では `⌘`、Windows/Linux では `Ctrl` を表示します。

## ショートカット一覧

### 分析・修正

| 操作 | Mac | Windows / Linux |
|------|-----|-----------------|
| 分析 & AI文体修正 | `⌘` `Enter` | `Ctrl` + `Enter` |
| テキスト分析のみ | `⌘` `⇧` `E` | `Ctrl` + `Shift` + `E` |
| AI文体修正のみ | `⌘` `⇧` `J` | `Ctrl` + `Shift` + `J` |

### 編集

| 操作 | Mac | Windows / Linux |
|------|-----|-----------------|
| Undo（元に戻す） | `⌘` `Z` | `Ctrl` + `Z` |
| Redo（やり直し） | `⌘` `⇧` `Z` | `Ctrl` + `Shift` + `Z` |

## Undo / Redo の対象

以下の操作が Undo / Redo の対象になります（最大 30 段階）。

- テキスト入力（1秒間隔でスナップショット）
- 提案の「承認」（テキスト置換）
- AI文体修正の「適用して置き換え」
- サンプルテキストの読み込み
- テキストのリセット

## 無効になる条件

| 状況 | 分析系ショートカット | Undo / Redo |
|------|---------------------|-------------|
| IME 変換中 | 無効 | 無効 |
| `input` / `textarea` 入力中 | 無効 | 無効 |
| エディタ（contentEditable）入力中 | **有効** | **有効** |

## カスタマイズ

設定ダイアログ（API Keys）内の「ショートカットキー」セクションから、各アクションのキーバインドを変更できます。

1. 変更したいアクションのキー表示ボタンをクリック
2. 「キーを押してください...」と表示されたら、新しいキーコンビネーションを押す
3. 設定は `localStorage` に自動保存され、ブラウザを閉じても保持
4. 「デフォルトに戻す」で初期設定にリセット

- Esc で記録キャンセル
- 他のアクションと重複するキーは割り当て不可（アラートで通知）

## 技術仕様

- OS 判定: `navigator.userAgent` でMac/Win/Linuxを検出（`src/utils/platform.js`）
- キー名は変数化されており、ハードコードなし
- モディファイアキー判定: Mac は `metaKey`、Windows は `ctrlKey`
- ショートカット表示はボタンUI上にOS適応で自動表示
- Undo履歴: HTMLスナップショット方式、`useUndoRedo` カスタムフック（`src/hooks/useUndoRedo.js`）

## テスト手順

1. エディタにテキストを入力
2. `⌘+Enter`（または `Ctrl+Enter`）→ 分析 & リライトが同時実行されること
3. 提案の「承認」を押す → テキストが変わること
4. `⌘+Z` → 承認前の状態に戻ること
5. `⌘+⇧+Z` → 再度承認後の状態になること
6. `⌘+⇧+E` → 分析のみ実行（リライトダイアログなし）
7. `⌘+⇧+J` → リライトのみ実行（提案パネルクリア）
8. IME 変換中にショートカット → 何も起きないこと
9. API キー入力欄にフォーカス中に `⌘+Enter` → 何も起きないこと
